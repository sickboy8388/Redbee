## How to contribute

1. **Fork** the repo and create a branch: `git checkout -b feat/my-feature`
2. Make your changes
3. **Test** that everything works
4. Open a **Pull Request** with a clear description

## What is welcome

- New attack techniques documented with working commands
- Automation scripts (Python, Bash)
- Fixes to outdated or broken commands
- Improvements to the interactive HTML guide
- New API-specific wordlists
- Translations

## Standards

- Commands must be tested and working
- Every technique must reference OWASP/CWE
- No payloads targeting real unauthorized systems in script output
- Respect the legal disclaimer â€” authorized use only

## Issues

Open an issue for:
- Bugs in commands or scripts
- Suggestions for new techniques
- Clarification requests

*Remember: use these techniques only on systems you are authorized to test.*
